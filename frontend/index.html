How it works

Data integrity
by design.

Every piece of data passes through validation gates before it affects your dashboard or models.

01
Ingest from Social APIs
Live filtered streams from Twitter/X, Reddit, and Mastodon flow into a Redis buffer via authenticated API connections — secrets stored in environment variables, never code.
● Live stream active
02
Schema Validation (Great Expectations)
Every batch is validated against an expectation suite before processing. If the API adds or removes fields, the checkpoint fails and raises an alert — before any model sees bad data.
⚡ Schema drift alerting on
03
Bot Score Computation
Validated posts are scored by the detection engine: account age, post frequency (posts/hour), follower ratios, and cosine similarity against known bot text patterns.
✓ 94.7% accuracy
04
CI/CD Deploy Gate
GitHub Actions runs data validation tests, security scans (Bandit), and unit tests on every push. The deploy step only runs if all gates pass — schema changes block the pipeline.
✕ Fails on schema change
05
Dashboard + Drift Monitoring
Clean, scored data surfaces in the real-time dashboard. Prometheus exports custom accuracy metrics; Grafana alerts when model drift exceeds acceptable thresholds.
● Grafana watching
